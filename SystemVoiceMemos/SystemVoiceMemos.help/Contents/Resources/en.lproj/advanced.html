Advanced

File formats, storage locations, and automation

File Formats

M4A Format

SystemVoiceMemos saves recordings as M4A files:

  • Container: MPEG-4 Part 14 (.m4a)
  • Audio Codec: AAC (Advanced Audio Coding)
  • Bitrate: Variable, typically 256 kbps for high quality
  • Sample Rate: 48 kHz
  • Channels: Stereo (2 channels)

Why M4A?

M4A was chosen because it offers excellent audio quality at relatively small file sizes and is widely compatible across devices and software.

Compatibility

M4A files play natively on:

  • macOS (QuickTime, Music, TV apps)
  • iOS (Music, Voice Memos)
  • Windows (Windows Media Player, iTunes, VLC)
  • Linux (VLC, Audacity, most players)
  • Most smart TVs and media players

Storage Location

Default Path

Recordings are stored in the app's container:

~/Library/Containers/com.arambdev.SystemVoiceMemos/Data/Documents/Recordings/

Sandboxing

SystemVoiceMemos is sandboxed for security. This means:

  • All app data is contained within the app's sandbox
  • The app cannot access arbitrary files on your system
  • Files are isolated from other apps unless explicitly shared

Accessing Files

To work with recordings outside the app:

  • Show in Finder: Right-click any recording
  • Export: Drag recordings from Finder to any location
  • Open in: Choose "Open in QuickTime" or another app
Note: The sandbox location is backed up by Time Machine if you have that enabled. Ensure your backup includes your Home Library.

Database Structure

SwiftData

SystemVoiceMemos uses SwiftData for persistent storage of metadata:

  • RecordingEntity: Stores recording metadata (title, date, duration, file path)
  • FolderEntity: Stores folder information and relationships

File vs Database

Important distinction:

  • Audio files: The actual M4A audio data
  • Database: Metadata, folder associations, and app state

Deleting a recording removes both the database entry AND the audio file.

Automation and Scripting

AppleScript Support

Currently, SystemVoiceMemos has limited AppleScript support. Future versions may include:

  • Start/stop recording
  • Query recordings
  • Export recordings

Shortcuts Integration

macOS Shortcuts integration is planned for future updates, allowing you to:

  • Create shortcuts that start recording with one tap
  • Automate post-processing of recordings
  • Integrate with other apps in your workflow

Folder Actions

You can use macOS Folder Actions on the recordings directory to trigger workflows:

  1. Right-click the recordings folder in Finder
  2. Choose Services → Folder Actions Setup…
  3. Attach an AppleScript or Automator workflow
Caution: Be careful with folder actions that modify or move files, as this could break the link between the database and the actual audio files.

Multi-Track Recording Deep Dive

How It Works

When multi-track recording is enabled:

  • System audio and microphone are captured on separate tracks
  • Each track is encoded independently
  • Tracks are muxed into a single M4A container

Stem Export Details

Exporting stems creates separate files:

  • Each track is written to its own M4A file
  • Files are named with track suffixes (-system, -mic)
  • Original multi-track file remains unchanged

Post-Production

Exported stems can be:

  • Opened in DAWs (Logic Pro, GarageBand, Ableton Live, Reaper)
  • Mixed with different track balances
  • Processed with effects and plugins
  • Exported to other formats (WAV, MP3, etc.)

ScreenCaptureKit

Underlying Technology

SystemVoiceMemos uses Apple's ScreenCaptureKit framework:

  • Introduced in macOS 12.3
  • Replaces the deprecated CGDisplayStream API
  • Designed for screen recording and streaming apps
  • Provides efficient access to display and audio streams

Audio Capture

ScreenCaptureKit captures:

  • System audio from the specified display
  • Does NOT capture video (only audio stream is used)
  • Works even if no screen content is visibly changing
Note: ScreenCaptureKit requires macOS 12.3+, but SystemVoiceMemos requires macOS 15.0+ for other features and improvements.

Privacy and Security

Local-First Design

SystemVoiceMemos is designed with privacy in mind:

  • All recordings stored locally on your Mac
  • No cloud sync or upload features
  • No analytics or telemetry
  • No user tracking or identification

Open Source

The app is fully open source:

  • Source code available on GitHub
  • You can audit the code yourself
  • Community contributions welcome

Code Signing

Release builds are:

  • Signed with a Developer ID certificate
  • Notarized by Apple for macOS security
  • Stapled with the notarization ticket
Verification: You can verify the app's signature by running codesign -dv /Applications/SystemVoiceMemos.app in Terminal.

Building from Source

Requirements

  • Xcode 16 or later
  • macOS 15.0+ SDK
  • A Developer Team for code signing (optional for development builds)

Build Steps

  1. Clone the repository: git clone https://github.com/aramb-dev/SystemVoiceMemos.git
  2. Open SystemVoiceMemos.xcodeproj in Xcode
  3. Select the SystemVoiceMemos scheme
  4. Press Cmd+B to build or Cmd+R to run

See the GitHub repository for more detailed build instructions.
