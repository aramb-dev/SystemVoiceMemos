Troubleshooting

Common issues and solutions

Permission Issues

"Screen Recording permission not granted"

Symptoms: Recording won't start, error message about permissions

Solutions:

  1. Open System Settings → Privacy & Security → Screen Recording
  2. Find SystemVoiceMemos in the list
  3. Enable the toggle switch
  4. Quit and restart SystemVoiceMemos
Tip: macOS sometimes requires you to toggle Screen Recording permission off and on again after an app update. If permission was already granted but stopped working, try toggling it.

Microphone permission denied

Symptoms: Can't enable microphone recording, "Settings" button appears

Solutions:

  1. Open System Settings → Privacy & Security → Microphone
  2. Find SystemVoiceMemos and enable the toggle
  3. Restart the app

Previously denied permissions

If you previously clicked "Don't Allow" for a permission:

  1. Open System Settings and manually enable the permission as described above
  2. The app won't prompt again until you grant permission in System Settings

Recording Issues

"Cannot start recording" error

Possible causes and solutions:

  • Another app is using system audio: Quit screen recorders, audio routers, or other capture apps
  • Permission just granted: Restart SystemVoiceMemos after granting Screen Recording permission
  • Display sleep issue: Prevent your Mac from sleeping during long recordings

Recording has no audio

Check these:

  • Is audio actually playing on your Mac? Check system volume
  • Are headphones connected? Audio may be routing there instead of to the system output
  • Try playing from a different app (Music, YouTube, etc.)
  • Open the recording file in QuickTime to verify it truly has no audio

Recording is too quiet or distorted

Solutions:

  • Too quiet: Increase the source volume before recording
  • Distorted: Lower the source volume — capture at moderate levels
  • Check system effects: Disable any equalizers or audio enhancers in System Settings

Recording stops unexpectedly

Possible causes:

  • Disk space: Ensure your Mac has sufficient free disk space
  • System sleep: Prevent display sleep or use a caffeinate tool
  • App crash: Check Console.app for crash logs and report the issue

Playback Issues

No sound during playback

Check:

  • System volume is not muted and is at a reasonable level
  • Audio output is set to the correct device (check in System Settings → Sound)
  • The recording file actually contains audio (open in QuickTime)

Playback is choppy or stutters

Solutions:

  • Close other apps that may be using significant CPU
  • Check if the recording file is corrupted (try opening in QuickTime)
  • Verify the file is on local storage (not a network drive)

Can't select or play a recording

Solutions:

  • Verify the recording file still exists in the recordings directory
  • Restart SystemVoiceMemos if the UI seems unresponsive
  • Check Console.app for any error messages

Update Issues

Updates not downloading

Check:

  • Internet connection is active
  • Firewall isn't blocking connections to github.com
  • The appcast URL is correct: https://raw.githubusercontent.com/aramb-dev/SystemVoiceMemos/main/appcast.xml

Update fails to install

Solutions:

  • Ensure the app is in your Applications folder
  • Check that you have write permissions for the app
  • Try downloading manually from GitHub and replacing the app manually

"App is damaged" error after update

This is a macOS security issue. Solutions:

  1. Right-click (or Control-click) the app and choose Open
  2. Click Open in the security dialog
  3. The app is now trusted and will open normally

File and Folder Issues

Can't find recordings in Finder

Default location:

~/Library/Containers/com.arambdev.SystemVoiceMemos/Data/Documents/Recordings/

To reveal: Right-click any recording and choose Show in Finder.

Folders disappeared

Solutions:

  • Toggle the sidebar (Cmd+Option+S) to ensure it's visible
  • Check if you're in a specific folder view — click "All Recordings"
  • Restart the app to refresh the database

Recording shows wrong date or name

This could indicate a database issue. Solutions:

  • Right-click the recording and choose Show in Finder to verify the actual file date
  • Rename the recording if needed
  • For persistent issues, consider quitting the app and restarting it

Performance Issues

App is slow or unresponsive

Check:

  • Number of recordings — very large libraries (1000+) may be slower
  • Available RAM — close other apps if memory is low
  • Disk space — ensure your Mac has free space for the database to operate

High CPU or memory usage

During recording: Some CPU usage is normal, especially with multi-track recording.

When idle: Should be minimal. If not:

  • Check for background sync or indexing activity
  • Restart the app
  • Check Console.app for error messages

Getting Help

If you're still experiencing issues:

  • Report issues: GitHub Issues
  • Check for updates: Make sure you're on the latest version
  • Include details: macOS version, app version, steps to reproduce, and any error messages
Privacy Note: When reporting issues, avoid sharing sensitive information about your recordings or personal data. Focus on the technical details of the problem.
